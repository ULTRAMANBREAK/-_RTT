# 硬件连接总表

> 适用项目：STM32F407-RT-SPARK 外卖箱端。本文档记录当前代码和 CubeMX 配置中的实际硬件占用，接线和联调优先以这里为准。

## 1. 串口连接

| 功能 | RT-Thread 设备 | STM32 引脚 | 外设连接 | 参数 | 备注 |
| --- | --- | --- | --- | --- | --- |
| ESP32 上位机/联网/MQTT 桥 | `uart3` | `PB10=USART3_TX`, `PB11=USART3_RX` | ESP32 `G8=TX -> PB11/RX`, ESP32 `G17=RX <- PB10/TX` | 115200, 8N1, 无流控 | 每条消息一行 JSON，以 `\n` 或 `\r\n` 结尾 |
| GPS/ATGM336H | `uart2` | `PA2=USART2_TX`, `PA3=USART2_RX` | GPS TX 接 `PA3/RX`，GPS RX 接 `PA2/TX` | 由 GPS 模块代码配置 | STM32 只使用 GPS 速度，不使用坐标业务输出 |
| UART4 雷达/NFC 复用总线 | `uart4` | `PA0=UART4_TX`, `PA1=UART4_RX` | 接四路串口扩展/复用模块公共串口 | 115200, DMA RX | `uart4_app` 独占 `uart4`，不要再让 RC522/LD2451 单独打开 UART4 |

## 2. UART4 复用通道

选择线：

| 选择线 | STM32 引脚 | 宏 |
| --- | --- | --- |
| B | `PD10` | `APP_UART4_MUX_PIN_B` |
| A | `PD9` | `APP_UART4_MUX_PIN_A` |

通道定义：

| 通道 | BA | 外设 | 业务含义 | 处理逻辑 |
| --- | --- | --- | --- | --- |
| `ch0` | `00` | LD2451 雷达 | 左雷达 | 距离异常驱动左侧 WS2811，`PE2` |
| `ch1` | `01` | LD2451 雷达 | 中间雷达 | 距离异常驱动左右两侧 WS2811，`PE2 + PE5` |
| `ch2` | `10` | LD2451 雷达 | 右雷达 | 距离异常驱动右侧 WS2811，`PE5` |
| `ch3` | `11` | RC522/RFID 串口读卡器 | NFC 夹子/解锁卡 | 取餐、送达、解锁卡识别 |

模式切换测试按键：

| 功能 | STM32 引脚 | 有效电平 | 用途 |
| --- | --- | --- | --- |
| 到达/行驶模式切换 | `PC0` | 低电平有效 | 当前用于测试：行驶模式轮询三路雷达，到达模式只轮询 NFC |

注意：`PD9/PD10` 不能被 FSMC/LCD 重新占用。CubeMX 重新生成后必须检查 GPIOD FSMC 初始化和反初始化掩码。

## 3. NRF24L01 夹子 LED 命令链路

| 功能 | STM32 引脚/设备 | 宏/配置 | 备注 |
| --- | --- | --- | --- |
| SPI 总线 | `SPI1` | `APP_NRF24_SPI_BUS_NAME="spi1"` | 数据线使用 `PA5/PA6/PA7` |
| SCK | `PA5=SPI1_SCK` | CubeMX SPI1 |  |
| MISO | `PA6=SPI1_MISO` | CubeMX SPI1 |  |
| MOSI | `PA7=SPI1_MOSI` | CubeMX SPI1 |  |
| CSN | `PA4` GPIO | `APP_NRF24_CSN_PIN=GPIO_PIN_4`，设备名 `spi10` | `PA4` 是软件片选 GPIO，不要配置成 `SPI1_NSS` |
| CE | `PE3` GPIO | `APP_NRF24_CE_PIN=GET_PIN(E, 3)` | CE 必须独立于 CSN |
| RF 信道 | 40 | `APP_NRF24_CLIP_RF_CHANNEL` | 夹子端需要一致 |
| Payload | 1 byte | `APP_NRF24_CLIP_PAYLOAD_SIZE` | 命令字：off/on/pulse/blink/toggle |

夹子 LED 命令不是 WS2811 灯组。ESP32 发 `{"type":"blink","cardId":"01"}` 后，STM32 通过 NRF24 控制对应夹子 LED。

## 4. WS2811 车身警示灯

| 位置 | STM32 引脚 | LED 数量 | 颜色业务 |
| --- | --- | --- | --- |
| 左侧 WS2811 | `PE2` | 59 | 雷达距离警告用红色，左转用蓝色；WS2811 实测字节顺序为 RBG |
| 右侧 WS2811 | `PE5` | 50 | 雷达距离警告用红色，右转用蓝色；WS2811 实测字节顺序为 RBG |

雷达映射：

| 雷达通道 | 灯组响应 |
| --- | --- |
| `ch0` 左雷达 | 左侧 `PE2` |
| `ch1` 中间雷达 | 左右两侧 `PE2 + PE5` |
| `ch2` 右雷达 | 右侧 `PE5` |

距离阈值默认 `5m`。短时间无数据保持上一次风险状态，长时间无数据解除报警。

## 8. ESP32 串口协议要点

ESP32 到 STM32：

```json
{"type":"req","want":"speed"}
{"type":"req","want":"temp"}
{"type":"arrived","cardId":"01"}
{"type":"blink","cardId":"01"}
```

STM32 到 ESP32：

```json
{"type":"speed","speed":12.5}
{"type":"temp","FL":62.3,"FR":61.8,"BL":60.5,"BR":61.2}
{"type":"nfc","action":"BIND","cardId":"01"}
{"type":"nfc","action":"UNBIND","cardId":"01"}
{"type":"unlock"}
{"type":"fall"}
{"type":"theft"}
```

`cardId` 必须是两位字符串：`"01"` 到 `"04"`。

## 9. 已知冲突和复核点

- `PA4` 是 NRF24 的 GPIO CSN，不能被 CubeMX 生成成 `SPI1_NSS`。
- `PE2/PE5` 是 WS2811，不能拿去做 NRF24 CE 或其他普通 GPIO。
- `PD9/PD10` 是 UART4 复用选择线，不能被 FSMC/LCD 抢走。
- `uart4` 只能由 `uart4_app` 统一打开，RC522/LD2451 不能各自独占打开。
- STM32 端已经不负责 WiFi/MQTT，ESP32 负责联网和 MQTT。
- CubeMX 重新生成后，重点检查：`PA4`、`PD9/PD10`、`PF9 LCD backlight PWM`、`PB0 buzzer`。
